Wikipedia:Bug reports

From Simple English Wikipedia, the free encyclopedia


The MediaWiki software uses the SourceForge trackers to track bug reports and feature requests.

Go to meta:MediaWiki feature requests and bug reports to talk about bugs you find and features you would like.


Don't report errors in articles. Just click edit this page and make the change yourself. Or write it on that article's talk page.

[edit] Reported bugs

[edit] Forced lowercase

Currently, MonoBook skin (although very beautiful) nonsensically forces some UI stuff, like MediaWiki:Mycontris, to be all in lowercase like it looks sexy or something. It is bad English and is misleading. --Menchi (Talk). 12:14, 29 May 2004 (UTC)

Unless there is another MediaWiki message that uses lowercase I. --Menchi (Talk). 12:23, 29 May 2004 (UTC)
Eeek. I've reported this at Meta. It's a problem for some other languages as well it seems. Angela 03:42, 30 May 2004 (UTC)
Adding li { text-transform: none; } to MediaWiki:Monobook.css fixed this. Angela 14:54, 24 Jun 2004 (UTC)

[edit] The $1 search

Currently, if I search for anything in the Simple English Wikipedia search box, the results page includes the following statement: "For more information about searching Wikipedia, see $1." See $1? I'm guessing that's a missing text string or something, but it could be very confusing to users of the Simple English Wikipedia. Starblind 14:41, 14 Jun 2005 (UTC)

[edit] Non-existent categories

If I add a category to an article and it doesn't exist, it doesn't go red anymore, it pretends to exist. It's getting annoying. Archer7 19:05, 25 February 2006 (UTC)

Appears to be fixed. Archer7 10:35, 26 February 2006 (UTC)

[edit] Screwed up dates

Some of the dates in signatures have changed years, some even to March 2006. Archer7 22:46, 2 March 2006 (UTC)

Oh, it is March 2006. But the message said 16th March 2006, when it was 16th March 2005. Archer7 22:47, 2 March 2006 (UTC)
