[edit] Basic English Bot (request)

Meta bot request

I'm on Wikipedia as SBHarris, and just discovered Simple English Wikipedia. Kick me if this has been suggested someplace else, but it occurs to me that what Simple English Wikipedia needs, is a lot of robotic flagging of articles to see which of their component words are part of the Basic English 850, or 1500, or derivatives thereof (a much longer list). The output might give words in three different colors, for example.

It would be most useful if a user had some way of running the `bot repetitively on an article, while editing it. The best place for the bot to run in the background, would obviously be to have it run every time the user did a "show preview." Results would come up very much like the "linked" or "unlinked" colors now used for THOSE purposes.

For this reason, such a "bot" would really ideally be part of the mediawiki software that runs this place. So this would more than a bot, but change in the basic software. But anything (even a bot which runs once a day and checks all articles) would help.

Without a lot of experience, it's quite difficult for the average native English speaker to tell if words below a certain complexity are part of the Basic English 850 or 1500 list. However, with these things automatically flagged by bot, it IS easy, even for inexperienced people to spot target words outside basic English, and then to make a decision if there is some shorter and more basic replacement that works as well, or if the word needs to stand "as is."

Won't one of your considerate 'bot makers (or you software mogels who did the original programming for this Version) consider working on this? It really would change everything here.

Addendum: I can think of more complex stuff, and most of these ideas apparently aren't as creative as I thought, since a friend of mine also came up with most of them, even as I was explaining the problem, before I'd gotten to my solutions. (So something of this sort has got to happen here eventually-- it's only a matter of WHEN, not IF). Here's a related Bot solution: There is need for a Reverse Thesaurus Bot, which is a normal thesaurus but with the Basic English word or phrase as a "redirect". This would not only flag words as not being part of BE 850 or BE 1500, but then also, for lots of words, might be able to give the Basic English word or phrase which is the suggested replacement. There is probaby such a thing already in print form, for writers of Basic English literature. But the process needs badly to be automated here, and probably could be. Here if nowhere else!

Sharris 20:56, 8 October 2006 (UTC) (SBHarris on standard Wikipedia).

I have an idea. Rather than a bot that no-one will use, why don't we just set up a web-based Java system. You type the article name in the box, it fetches it and does it with no installs. It's quite a basic system, but I'm not sure if I can do it. Archer7 - talk 21:44, 8 October 2006 (UTC)
